Exordio

Este proyecto es una rara avis en la obra de escrivivir.co: se ocupa del negocio — las antípodas del ocio, nuestra demarcación natural. Pero uno no puede escrivivir desde palacios de marfil sino manchado.

La incursión SDK es nuestra forma de pisar territorios ajenos: generando cajas de herramientas. Si sobre la mesa está el negocio como usura sobre relaciones de deuda creando dinero de la nada, el préstamo al 0 % con cláusulas de interés negativo es la fórmula limpia que pasa bajo el radar. Identificada, pero como elemento que no produce "eso" por lo que el Estado y Hacienda tienen su red de impuestos.

Dos ejes. Eco-ética: la riqueza circula cuidando la cadena completa. Eco-fraterna: el préstamo ofrece suelo, no sombra con guadaña. Un protocolo por el cual unas personas pueden involucrarse en una actividad de negocio afín a nuestra idea de ocio.

El doble eje

Eco-sostenible

El préstamo activa la economía con cuidado de la cadena completa. El activo no se adquiere por el canal más barato: pasa por las manos de un proveedor que trabaja en condiciones dignas, con materiales de cadena ética certificada. El dinero toca tierra, genera impacto, deja huella documental. La eco-sostenibilidad no es un adjetivo del activo: es la arquitectura del encargo.

Eco-fraterno

El préstamo ofrece suelo económico fraterno. El activo llega al receptor sin contador de intereses ni fecha perentoria: llega como infraestructura de agencia. El contrato formaliza ante la administración tributaria lo que en el fondo es un acto de fe: que el prestamista presta lo que está dispuesto a no recuperar, y que el receptor recibe algo que le pertenece de verdad aunque deba devolverlo si puede.

Protocolo

Cinco fases

#FaseProduce
1Comisionar el activoEncargo al proveedor con especificaciones éticas
2Documentar el activoFactura, certificado de trazabilidad, anexo técnico
3Formalizar el préstamoContrato con 4 cláusulas, documentación del receptor
4Acto de entregaEntrega física, acta firmada, cuadernillo al receptor
5Registro y seguimientoPresentación fiscal, archivo, registro de retornos
→ framework-5-fases.md

Cuatro cláusulas base

[01] Interés 0 % — el tiempo no penaliza al receptor.
[02] Revalorización invertida — la plusvalía beneficia al receptor.
[03] Condición-umbral de solvencia — no hay obligación bajo el umbral.
[04] Devolución parcial en especie — módulo a módulo, sin calendario.

→ clausulas-base.md

Cuatro escenarios de devolución

EscenarioResultado
AIntactoConserva el activo. Umbral no superado. Préstamo en suspenso.
BGradualDevuelve módulo a módulo según puede, al valor nominal.
CLiquidaciónVende módulos para necesidades vitales. Sin devolución.
DReboteUsa módulos como capital semilla, prospera, activa devolución.

Mods

Un mod es una instanciación del SDK para un activo, jurisdicción y circunstancia específicos. Reside en su propia rama (mods/{nombre}) y añade un directorio mod-{nombre}/ encima de la base.

Plan Oro rama: mods/oro

Sacralizar el dinero habida cuenta del albedo en la historia de este metal precioso. Hacer un rosario de oro hace del artefacto tanto una cuenta de equilibrio económico como la actividad propia de contar plegarias.

Crear un mod nuevo

$ git checkout -b mods/{nombre}            # rama desde main
$ mkdir mod-{nombre}
$ cp sdk/config.template.md \
     mod-{nombre}/proyecto.config.md        # rellenar {{PLACEHOLDERS}}
# instanciar plantillas de roles, bitácoras, contrato
# crear referencia de cadena ética del activo
# escribir mod-{nombre}/README.md
→ instrucciones completas en README → config.template.md

Don't repeat yourself

El SDK vive en main. Cada mod vive en su rama (mods/*). El mod hereda la base completa vía git — no duplica, no copia, no diverge. La regla: lo genérico se escribe una vez; lo específico se añade al lado.

$ git checkout mods/oro
$ git merge main

  → sdk/ se actualiza
  → mod-oro/ no se toca
  → cero conflictos

Cada mod añade sólo contenido nuevo. Cuando el SDK mejora en main, un git merge lo propaga a todos los mods sin que ninguno pierda su trabajo específico. Ficheros del SDK y ficheros del mod ocupan directorios disjuntos: no hay colisión posible.

Escribe una vez. Instancia muchas. Actualiza desde la raíz.
